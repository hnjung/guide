규칙

예약어

레이아웃

  • #wrap : 본 페이지와 팝업 페이지를 감싼 아이디
  • #page : 본 페이지
  • #header : 본 페이지 헤더
  • #main : 본 페이지 바디
  • .section : 콘텐츠 영역 대 그룹핑 {검색 / 등록폼}
  • .group : 콘텐츠 영역 소 그룹핑 {검색 / 필수입력폼 / 추가입력폼}
  • #footer : 본 페이지 푸터
  • #popup : 팝업 페이지

구조

  • .xx {.card} : 최상위 클래스로 하나의 단어
  • .xx-wrap {.card-wrap} : 한번 더 감싸야 할 때
  • .xx-head {.card-head} : 영역의 상단
  • .xx-list {.card-list} : 콘텐츠 목록 영역
  • .xx-item {.card-item} : 콘텐츠 목록의 Row Data
  • .xx-body {.card-body} : 콘텐츠 (목록 유형이 아닌 경우)
  • .xx-foot {.card-foot} : 영역의 하단
  • .xx.option-yy {.card.option-fixed} : xx에서 파생된 유형

이벤트

  • .is-xx {.is-active} : 스타일 변경을 스크립트로 조작하는 클래스
  • .fn-xx {.fn-popup-open} : 스타일 변경 없이 함수를 실행하기 위한 클래스

HTML

  • 영역 주석 : <!-- #header --><!--// #header -->, <!-- .group --><!--// .group -->
  • 날짜 추가 주석 : <!-- 2023-05-18 : 탭 활성화 코드 수정(구체적으로 작성) -->
  • 개발 참고 주석 : <!-- [D]클릭 시 .is-active 클래스 추가 -->
  • 들여쓰기 : 1탭(공백 4칸)
  • 특수기호 : Entity Code로 변환하여 작성

CSS

  • 미니마이징 시 주석은 모두 삭제되므로 자세하게 작성
  • 파일 수정 시 상단의 수정 주석에 날짜와 내용, 작성자 업데이트
  • 클래스명 : 하이픈{-}으로 구분하여 소문자로 작성
  • 문법 규칙 : .class {property: value; property2: url('./xxx/xxx.png');}
  • 속성 선언 순서 : display > visibility > position > z-index > top, right, bottom, left > overflow > float > width > height > margin > padding > background > border > align > font > color > cursor > transform > transition
  • 단위 : 모바일 ~ 태블릿은 {rem}단위(브라우저 사용자 설정의 폰트 크기 변경 후 화면 테스트 필요), PC는 {px}단위로 작성
  • 선택자 선언 시 3뎁스 이하로 종속하여 선언 : .xx .yy > strong {}
  • html에 인라인 스타일 금지, 가능한 모두 CSS 파일에 클래스 선언 후 작업 (가이드에 있는 인라인 스타일은 카피 후 삭제)
  • {*}선택자와 {!important}속성은 가급적 사용 금지

Javascript

  • Javascript ES6 문법 사용 권고
  • 중복되는 구문은 가능한 함수로 선언하여 작성
  • 공통변수 : commUiVar 객체에 선언
  • 함수명 : commUiFn으로 시작하고 단어 첫번째 글자는 대문자로 작성 - const commUiFnPopupOpen = () => {};
  • 매개변수명 : (_)으로 시작하여 작성 - const commUiFnPopupOpen = (_element, _index) => {};
